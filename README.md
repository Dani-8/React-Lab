# 🧪 React Practice Lab

A high-fidelity, interactive sandbox engineered to bridge the gap between **WRITING CODE** and **UNDERSTANDING STATE.** This lab serves as the technical engine for my comprehensive React Learning Journey, providing a visual playground for complex frontend patterns.


## 🏗️ Core Architecture

The project is built with a **Registry-Based Architecture**, enabling modular insertion of experiments without bloating the core application logic. Each module is **Dynamically Mapped**, ensuring high performance and scalability.


## 🔭 Key Innovations: The State Watcher

Unlike standard tutorials, this lab features a real-time **State Watcher**. Every interaction in the **Canvas** is reflected in a dedicated debugging panel, showing exactly how React handles **Data Persistence**, **Object Spreading**, and **Re-rendering Cycles**.


## 📚  Integrated Learning Views

Every experiment in the lab provides four distinct perspectives:

- **Preview**: Live interactive component for UI testing  
- **Theory**: Deep-dive documentation and concept breakdowns  
- **JSX**: Implementation-ready source code with syntax highlighting  
- **Architecture**: File-tree visualization for real-world project structure  


## 📂 Modules Covered

### 1. Fundamentals
- **Props Drilling vs. Composition**
- **Advanced Mapping & Key Optimization**

### 2. Deep Dive: Hooks
- **State**: useState (Primitives & Objects)  
- **Side Effects**: useEffect (Lifecycles & Cleanup)  
- **Memoization**: useMemo & useCallback  
- **DOM / Persistence**: useRef  
- **Context**: Global state without prop drilling  
- **Custom Hooks**: useFetch implementation  

### 3. State Management (Redux Toolkit)
- **Slice Architecture** (Auth, Cart, Counter)  
- **Async Logic** with RTK Query / Thunks  
- **Global State Synchronization**

### 4. Enterprise Routing (React Router 6.4+)
- **Dynamic & Nested Routes**  
- **Data Loaders & Error Boundaries**  
- **Layout Persistence Patterns**
